3.2 Describe ways to improve productivity and efficiency

Copius amounts of caffiene seem to help with 'productivity'.

There are still ways to improve the general efficiency and productivity of the project and the process overall. One example is explained partly beforehand, which is forming teams and developing in groups, with the benefits outlined. Being able to divide the labour between a larger party or even multiple parties in large products leaves each individual more time to focus on a specific task. They also allow us to reduce errors and increase functionality by combining experience and skill levels.

Another method of increasing the efficiency of website development is to use a framework, specifically a Web Development Framework, which is a pre-created document with HTML and CSS already in it, often with additional javascript and other coding languages too. They can fix problems with browser compatibility, have common functions baked into them and provide layouts to transfer content faster. [1]

Important to productivity and efficiency is the plan and specifically how each task and amount of time is handled. Often in web development, projects can have a very exhausting list of tasks and issues that need to be taken care of. Being able to properly prioritize and manage these tasks is very important to staying on schedule. Tasks generally should be run according to the amount of time they need allocated to be finished, although flexibility is important and being able to rank and prioritise tasks according to the situation is great for productivity. Important to this [idea is conceptualisation. Taking an idea and giving it a visual or physical form, even if a very basic one, and making the ideal result clear and achievable, gives developers something to aim towards and a better idea of how to get to that point.

3.4 Describe how you would go about testing digital solutions

I'm running out of witty things to say.

Digital solutions such as websites need to be properly tested before being given to a client or made public. There are quite a few methods and checks you can use to make sure that a product is suitably tested.

Some of the more obvious checks in terms of site content and navigation are at the top of the priority list. Images need to display as intended and not interfere with other parts of the page, written content should be free of spelling and grammar mistakes and there should be no dead ends (pages that don’t allow you to get out of them). Each object, element and function needs to work in isolation but also within the site as a whole, so essentially making sure they don’t make issues for each other. You’d go about testing this by using the website a lot and/or asking others to use it. Navigate to each and every of the website’s pages and use every function, in different orders and repeatedly. In terms of URLs, there are tools that exist to check websites for malfunctioning URLs and other links. You also need to check what happens when a site can’t find a specific page (error 404) and create a custom 404 page for the occasion if you can.

You would also have to check the compatibility and mobile-friendliness of a website. You’d have to make sure that the website functions properly on multiple web browsers as they do have differences in how they display and process information. Mobiles are the same again. To test it, you’d have to open the website on various platforms. Attempt to open the website on every major web browser (Chrome, Firefox, Safari etc). Also open it on mobile web browsers and on multiple screen sizes and orientations if possible. Additionally for mobile, if possible test it on both iOS and Android. [3]
